# NixOS Development Context

## Flake Patterns
- All inputs should follow nixpkgs for consistency
- New inputs must be added to both `inputs` and the `outputs` function parameters
- Run `nix flake check` after modifying flake.nix
- Use `nix flake update` to bump lock file; `nix flake lock --update-input <name>` for a single input

## NixOS Module Editing
- System-wide packages: `environment.systemPackages`
- Changes require `sudo nixos-rebuild switch --flake .` (tell user to run `./rebuild-nixos`)
- `hardware-configuration.nix` is auto-generated by `nixos-generate-config` — avoid editing it directly
- Module options: use MCP-NixOS server (configured in `.mcp.json`) to query 22K+ NixOS options

## Home Manager
- User programs: `programs.<name>.enable = true`
- User packages: `home.packages`
- Home Manager options: query via MCP-NixOS (4K+ options available)

## Audio / PipeWire / Bluetooth
- WirePlumber handles audio routing for bluetooth devices
- Known fix: disable auto headset profile in WirePlumber to prevent audio issues with AirPods
- Bluetooth config: `hardware.bluetooth.settings`
- Relevant files: `modules/home-manager/audio/pipewire.nix`, `hosts/nixos/hardware-configuration.nix`

## MCP Server Development
- Primary transport: stdio (spawned by Claude Code via `.mcp.json`)
- Optional HTTP mode for debugging/testing/external sharing
- Session management: TTL caching for auth tokens, API responses
- Development: `npm run dev` or configure in `.mcp.json`
- Testing: start server manually, use MCP Inspector for debugging

## CI/CD Patterns
- GitHub Actions with Workload Identity Federation (WIF) for cloud auth
- Nix flake checks run in CI for reproducibility
- Deploy: `gh workflow run <workflow>` for manual triggers
- Monitor: `gh pr checks` for CI status

## Package Search
- Use MCP-NixOS to query 130K+ packages in real-time rather than guessing package names
